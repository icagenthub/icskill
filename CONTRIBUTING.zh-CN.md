# 贡献指南

## Skill 收录原则

ICSkill 对收录的每个 Skill 执行安全审查。审查通过后方可合入。

### 提交要求

1. 每个 Skill 独立放在 `skills/<skill-name>/` 目录下
2. 必须包含 `SKILL.md`，声明：
   - 适用场景
   - 执行步骤
   - 数据访问范围（读取哪些文件/目录）
   - 工具调用范围（执行哪些命令、是否有网络请求）
   - 输出格式和边界
3. 如引入自外部来源，需附带 `ORIGIN.md` 说明来源、版本和上游地址
4. 脚本代码需可读、无混淆

### 安全审查

所有 Skill 在合入前需通过安全审查。审查由维护团队执行，具体标准不公开。

审查结果：
- **通过**：合入主分支
- **需修改**：PR 中标注修改要求
- **拒绝**：不符合收录标准

### 安全分级

每个通过审查的 Skill 会标注安全等级：
- `safe` — 纯读取和分析，不执行命令、不写文件
- `controlled` — 有命令执行或文件写入，但约束明确
- `review-required` — 涉及高权限操作，建议每次使用时人工确认

### 更新策略

引入自外部的 Skill 不自动同步上游。每次更新需重新提交 PR 并通过安全审查。
